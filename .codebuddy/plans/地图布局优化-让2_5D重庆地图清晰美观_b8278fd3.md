## Product Overview

对现有2.5D重庆城市地图（Three.js构建）进行布局重构优化，解决当前地图"太杂乱、不像正经地图"的问题。核心目标是大幅降低视觉密度、增加留白呼吸空间、让每个区域有清晰的视觉层次和辨识度，同时保持4个情绪区域（park/warm/active/calm）的交互兼容性。

## Core Features

- **精简前景建筑群**：将26栋前景楼（左侧14+右侧12）缩减至8-10栋，增大间距，让边缘不再拥挤
- **重构洪崖洞街区**：将13个紧密堆积的warmBlocks缩减至6-7个，采用错落有致的2-3层阶梯式布局而非密集网格
- **精简背景天际线**：将23栋背景塔楼缩减至10-12栋，拉开间距形成剪影层次
- **移除视觉噪音元素**：删除connectorSpine连接道路（4段）、plotAnchors地块标记（4组），简化洪崖洞步道层级
- **优化相机与雾效**：微调相机角度拉远视野，增强雾气浓度以增加景深和层次感
- **保持交互兼容**：RegionSensors位置不变、PetRouteLibrary路线点不变，宠物点击/悬停/巡游功能正常工作

## Tech Stack

- Three.js r128 (已有CDN引入)
- 原生 JavaScript ES6+
- Canvas 2D API (用于纹理/标签生成)

## Implementation Approach

### 核心策略：做减法 - 留白优先

整体思路不是重写，而是**系统性削减**各层物体数量并增大间距。具体策略：

1. **前景建筑（createAmbientProps）**：左侧14栋减至5栋（间距从0.85增至2.0+），右侧12栋减至4栋（间距从0.74增至2.5+）
2. **洪崖洞暖色街区（createHongyadongDistrict）**：13个block减至6-7个，改为2行x3列错落排列（底层3个较大、上层3-4个较小递进），删除topWalk步道层，保留boardwalk主步道和stairBridge楼梯
3. **背景天际线（createBackdropCity）**：leftBands 9栋减至4栋，middleBands 8栋减至4栋，rightBands 6栋减至3栋，总计11栋
4. **地形平台中的连接路（createTerrainPlatforms中connectorSpine）**：完全删除4段斜向道路
5. **地块标记（createPlotAnchors）**：完全删除4组plotRect
6. **相机与雾效**：相机Z轴后移2-3单位，fog密度从0.0072增至0.009
7. **洪崖洞支撑柱**：12根减至6根
8. **灯笼**：9个减至5个

### 性能影响

- 物体总数减少约60%，显著降低draw call和内存占用
- 雾效增强可进一步裁剪远处物体渲染
- 预期帧率提升明显（尤其低性能设备）

## Implementation Details

### 关键约束

- `RegionSensors` 的4个区域感应器坐标**不可改动**（否则宠物点击检测失效）
- `PetRouteLibrary` 中4条路线的点坐标**保持不动**（宠物巡游路径依赖这些坐标）
- 所有已有的工具函数（createWarmShophouse/createForegroundTower/createSkylineTower等）复用不修改

### 修改函数清单

| 函数 | 修改内容 | 改动幅度 |
| --- | --- | --- |
| setupCamera() | position.set z+2.5, lookAt微调 | 小 |
| initChongqingCity() | fog密度 0.0072->0.009 | 小 |
| createBackdropCity() | 三组bands各删减约50% | 中 |
| createAmbientProps() | 左14->5, 右12->4, 间距加大 | 大 |
| createTerrainPlatforms() | 删除connectorSpine整个Group | 中 |
| createPlotAnchors() | 函数体清空或直接return | 小 |
| createHongyadongDistrict() | warmBlocks 13->6-7, 删topWalk, 减少支柱灯笼 | 大 |
| createBridges() | 可能微调桥体宽度使其更纤细 | 可选 |
